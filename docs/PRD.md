# Career FIT OS — 제품 사양서 (PRD)

> 이 문서는 Career FIT OS의 현재 제품 사양서다.
> **현재 단계: 소규모 멀티유저 테스트** (사용자 각자 API 키 입력, 판단 기준 사용자별 설정).
> 1인용 시절 원본 사양은 `archive/career_fit_prompt.md` 참고.

**버전:** v2 (멀티유저 전환) · 2026-05-26

---

## 한 줄 목표

"나와 이 회사가 맞는지 빠르게 판단해서 시간을 아끼는 것"

## 대상 사용자

이직을 준비하는 사람. 본인 경력과 판단 기준을 설정하면,
공고를 넣을 때마다 판정 → 어필 포인트 → 이력서 초안이 자동으로 나온다.

> 1인용에서 출발했으나, MVP 검증 후 여러 사용자가 각자 쓸 수 있도록 확장 중.
> 각 사용자는 자기 경력·기준·API 키·데이터를 독립적으로 가진다.

## 핵심 철학 (코드 전체에 녹아야 함)

- 나도 회사를 고른다
- 서로 맞아야 오래간다
- 최종 결정은 내가 한다

판정이 "패스"여도 사용자가 "그래도 지원"을 선택할 수 있다.
OS는 판단을 돕고, 결정을 막지 않는다.

## 멀티유저 요구사항 (v2 신규)

### 인증 / 로그인
- 소규모 테스트에 맞는 가벼운 인증 (이메일 또는 초대 코드 수준).
- 거창한 권한 관리·관리자 대시보드는 이 단계에서 만들지 않는다.

### API 키 (사용자 각자 입력)
- 사용자는 본인 Claude API 키를 직접 입력한다.
- 서비스가 키를 대신 관리하거나 비용을 부담하지 않는다 (결제 시스템 없음).
- 키는 사용자별로 안전하게 보관. 평문 노출·콘솔 로그·타 사용자 노출 금지.

### 데이터 분리
- 한 사용자의 프로필·히스토리가 다른 사용자에게 보이면 안 된다.
- 기존 localStorage 단일 저장 → 사용자별 분리 저장으로 전환.
- 저장 방식(DB 종류 등)은 진행하며 결정. (DECISIONS.md 참조)

### 판단 기준 (사용자별 설정)
- 직무 적합 / 환경 적합 / 성장 적합 3개는 **기본 템플릿**.
- 사용자가 온보딩에서 자기 기준으로 수정 가능해야 함.
- 특정 산업(K-뷰티 등)에 기준을 하드코딩하지 않는다.

## 화면 구조 (3탭)

| 탭 | 역할 |
|----|------|
| 프로필 | 내 경력 + 판단 기준 설정 (사용자별, 1회) |
| 공고 판단 | 공고 넣으면 판정 + 어필 포인트 + 이력서 초안 자동 생성 |
| 지원 현황 | 분석한 공고 상태 관리 (지원 전 / 서류 / 면접 / 최종 / 패스) |

## Tab 1 · 프로필

이직 이유 선택 + 이력서 입력(텍스트/파일/링크) → AI가 분석해
포지셔닝 한 줄 + 핵심 강점 3개 + Must 조건 3개 + 판단 기준 생성.
사용자가 결과를 수정 가능. 사용자별로 저장.

## Tab 2 · 공고 판단

### 입력
- 이미지 업로드 (메인) — 잡코리아 등 복사 안 되는 공고. base64 → Claude 비전.
- 텍스트 붙여넣기 (보조) — 사람인·링크드인 등.

### 실행 순서 (오케스트레이션 — 바꾸지 말 것)
JD 입력 → 핵심 기준 체크(3개) → 판정 → 어필 포인트 → 이력서 가이드 → 이력서 초안

- 핵심 기준 체크: 각 항목 적합 / 부분 적합 / 미적합
- 판정: 지원 추천 / 검토 필요 / 패스 + FIT 점수
- 어필 포인트 3개 (개별 복사 가능)
- 이력서 초안: 자기소개 + 핵심 경력 3줄 + 지원 동기

> 1인용 시절 '화장품 경력 필수 감지'는 산업 중립적으로 일반화하거나
> 사용자가 켜고 끌 수 있는 옵션으로 전환.

### 판정별 다음 행동
- 지원 추천: 어필 포인트 전체 복사 / 새 JD 분석
- 검토 필요: 어필 포인트 미리 복사 / 새 JD 분석
- 패스: 새 JD 분석 / "그래도 지원할게요" 링크

## Tab 3 · 지원 현황

분석한 공고를 보드로 관리. 사용자별 분리.
- 필터: 지원 추천 / 검토 필요 / 패스 / 전체 (건수 표시)
- 컬럼: 회사명 / 직무 / FIT / 판정 / 상태 / 날짜
- 상태 드롭다운: 지원 전 / 서류 / 면접 / 최종 / 패스
