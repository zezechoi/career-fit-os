## 한 줄 목표

"나와 이 회사가 맞는지 빠르게 판단해서 시간을 아끼는 것"

## 고객

"경력은 있는데, 다음 자리는 신중하게 고르고 싶은 사람" — 이직자·중고신입·복귀자.
수백 개 공고 중 **내 기준**에 맞는 곳을 빠르게 걸러서, 이력서 커스텀까지 짧게 끝내는 게 목표.

> 출발은 제작자(최지예) 1인용(K-뷰티 이직)이었고 MVP까지 그 기준으로 완성했다.
> 2026-05 이후 **최소 멀티유저**로 확장 — 판단 기준을 하드코딩하지 않고,
> 사용자가 온보딩에서 자기 기준(직무·환경·성장)을 직접 생성·수정한다.
> 데이터는 브라우저별 localStorage로 분리(소규모 테스트 단계).

## 철학 (코드 전체에 녹아 있어야 함)

- 나도 회사를 고른다
- 서로 맞아야 오래간다
- 최종 결정은 내가 한다

판정이 부정적이어도 사용자가 "그래도 지원"을 선택할 수 있다.
OS는 판단을 돕고, 결정을 막지 않는다.

## 전체 구조

```
프로필 1회 설정 (경력 입력 → AI가 내 판단 기준 추출 → 수정/확정)
↓
공고(JD) 입력 — 이미지 또는 텍스트
↓
AI 분석(내 기준으로) — 핵심 기준 체크 → 필수 요건 감지 → 판정 → 어필 → 이력서 가이드
↓
이력서 초안 자동 생성
↓
지원 현황 보드에서 상태 관리
```

## 탭 구성 (3탭)

| 탭 | 역할 |
|----|------|
| 프로필 | 내 경력 + 판단 기준 설정 (한 번만) |
| 공고 판단 | 공고 넣으면 판정 + 어필 포인트 + 이력서 초안 자동 생성 |
| 지원 현황 | 분석한 공고 상태 관리 |

---

## Tab 1 · 프로필

이력서(텍스트/파일/링크)와 이직 이유를 1회 입력하면, AI가 **내 판단 기준**을 추출한다. 이후 공고만 넣으면 됨.

**온보딩이 추출하는 것 (결과 화면에서 모두 수정 가능):**
- 포지셔닝(summary) — 한 줄 정체성
- 커리어 목표(goal) — 다음 직장에서 이루고 싶은 방향. 모든 판정의 기준선
- 핵심 강점(strengths) / Must 조건(must) / 역량 키워드(keywords)
- **3축 판단 기준(criteria)** — 직무/환경/성장 각각이 "나에게" 무엇을 의미하는지(def)와 판단 근거(basis). 내 경력·직군에 맞게 개인화
- 어필 우선순위(appealPriority) — 공고에 지원할 때 앞세울 순서 3개

> 3축의 key(job/env/growth)·라벨·비중은 고정 템플릿이고, "의미(def/basis)"만 사용자별로 개인화된다.
> 특정 산업을 가정하지 않는다. 기존 프로필(개인화 기준 없음)은 재온보딩 전까지 범용 기준으로 폴백.

## Tab 2 · 공고 판단

### JD 입력
- **이미지 업로드 (메인)** — 잡코리아 등 복사 안 되는 공고 대응. base64 → Claude 비전
- **텍스트 붙여넣기 (보조)** — 사람인·링크드인 등 복사 가능 공고

### 분석 결과 구성 (실행 순서 = 오케스트레이션)

1. **핵심 기준 체크 (3개)** — 각 항목 0~100 점수 + 출처(JD 기반/프로필 기반/추정) + 근거. **내 프로필의 기준 정의(def/basis)로 채점**
   - 직무 적합 — 내 경력이 이 역할을 실제로 수행할 수 있는가
   - 환경 적합 — 이 회사의 규모·문화가 내 Must 조건과 맞는가
   - 성장 적합 — 이 포지션이 내가 원하는 경험을 줄 수 있는가
2. **필수 요건 감지 (산업 중립)** — [내 프로필] 대비 JD의 '필수 요건' 충족 여부 판정. 충족하기 어려운 필수 요건이 있으면(`mismatch`) 자동 경고 + 커버 전략 안내
3. **판정** — 지원 추천 / 검토 필요 / 패스
4. **어필 포인트 3개** — JD에 맞게 재구성된 강점 (개별 복사 가능)
5. **이력서 커스텀 가이드** — 한 줄 출력
6. **이력서 초안 자동 생성** — 자기소개 + 핵심 경력 3줄 + 지원 동기 (복붙 가능 수준)

### 판정별 다음 행동
- 지원 추천: 어필 포인트 전체 복사 / 새 JD 분석
- 검토 필요: 어필 포인트 미리 복사 / 새 JD 분석
- 패스: 새 JD 분석 / "그래도 지원할게요" 링크

## Tab 3 · 지원 현황 (CRM)

분석한 공고를 보드 형태로 관리.

**필터 탭** — 지원 추천 / 검토 필요 / 패스 / 전체 (각 건수 표시)

**컬럼** — 회사명 / 직무 / FIT / 판정 / 상태 / 날짜

**상태 드롭다운** — 지원 전 / 서류 / 면접 / 최종 / 패스

**메모** — 공고별 메모 추가

분석할 때마다 자동 저장.

## 하네스 + 오케스트레이션

이 서비스의 핵심 작동 원리.

**하네스 (말의 고삐)** — Claude가 딴 길로 안 가게 규칙 고정.
3축(직무·환경·성장)의 틀과 필수 요건 감지 규칙은 고정 + 각 축의 "의미"는 사용자 프로필에서 주입.
기준 틀이 고정돼야 공고 간 비교가 가능하고, 의미가 개인화돼야 사용자별 판정이 된다.

**오케스트레이션 (지휘)** — 할 일의 순서를 설계.
JD 입력 → 기준 체크 → 필수 요건 감지 → 판정 → 어필 → 이력서 가이드.
순서가 설계되면 공고만 넣어도 한 번에 결과가 나온다.

## 기술 스택

- 단일 HTML + Vanilla JS + Tailwind CDN
- Claude API (claude-sonnet-4-5) — **서버리스 프록시(`/api/claude`)를 통해서만** 호출. Anthropic 키는 서버 환경변수(`ANTHROPIC_API_KEY`)에만 존재, 프론트/브라우저에 절대 노출 안 됨
- 사용자 인증: 공유 비밀번호(`x-app-password` 헤더 → 서버 `APP_PASSWORD` 검증). 사용자는 키가 아니라 비밀번호만 입력
- localStorage 데이터 유지 (브라우저별 분리, 클라우드 동기화 off)
- 폰트 Pretendard
- **다크모드** (베이지 톤에서 판정 배지 구분이 약해 전환)

### localStorage 구조
```
careerfit_password  → 앱 접근용 공유 비밀번호 (Anthropic 키 아님)
careerfit_profile   → { reason, resume_raw, resume_sources, summary, goal,
                        strengths[], must[], keywords[],
                        criteria[{key, def, basis}],   // 3축 개인화 정의
                        appealPriority[], updated_at }
careerfit_history   → [{ id, company, position, verdict, requirement{level,note},
                        criteria[], appeals[], gaps[], ... , analyzed_at }]
```
