## 한 줄 목표

"나와 이 회사가 맞는지 빠르게 판단해서 시간을 아끼는 것"

## 고객

나(최지예) — 3개월 안에 K-뷰티 중소사로 이직해야 하는 사람.
수백 개 공고 중 내 기준에 맞는 곳을 빠르게 걸러서,
이력서 커스텀까지 30분 안에 끝내는 게 목표.

고객이 한 명(본인)이라 판단 기준·기능·사용 이유가 전부 명확하다.
범용이 아니라 1인용으로 좁힌 것이 핵심 설계 결정.

## 철학 (코드 전체에 녹아 있어야 함)

- 나도 회사를 고른다
- 서로 맞아야 오래간다
- 최종 결정은 내가 한다

판정이 부정적이어도 사용자가 "그래도 지원"을 선택할 수 있다.
OS는 판단을 돕고, 결정을 막지 않는다.

## 전체 구조

```
프로필 1회 설정 (경력 + 판단 기준)
↓
공고(JD) 입력 — 이미지 또는 텍스트
↓
AI 분석 — 핵심 기준 체크 → 화장품 경력 감지 → 판정 → 어필 → 이력서 가이드
↓
이력서 초안 자동 생성
↓
지원 현황 보드에서 상태 관리
```

## 탭 구성 (3탭)

| 탭 | 역할 |
|----|------|
| 프로필 | 내 경력 + 판단 기준 설정 (한 번만) |
| 공고 판단 | 공고 넣으면 판정 + 어필 포인트 + 이력서 초안 자동 생성 |
| 지원 현황 | 분석한 공고 상태 관리 |

---

## Tab 1 · 프로필

내 경력과 판단 기준을 1회 설정. 이후 공고만 넣으면 됨.

**현재 프로필 (최지예)**
- 경력: 해외영업 6년 + IT 마케터 1년
- 핵심 강점: 커뮤니케이션, 협상력, 스토리텔링
- 찾는 회사: 중소 K-뷰티, 해외영업 경력 살리면서 브랜드 전체를 볼 수 있는 곳
- 제약: 화장품 경력 없음 → 필수 요건이면 현실적으로 어려움
- 포지셔닝: K-뷰티 글로벌 오퍼레이터

---

## Tab 2 · 공고 판단

### JD 입력
- **이미지 업로드 (메인)** — 잡코리아 등 복사 안 되는 공고 대응. base64 → Claude 비전
- **텍스트 붙여넣기 (보조)** — 사람인·링크드인 등 복사 가능 공고

### 분석 결과 구성 (실행 순서 = 오케스트레이션)

1. **핵심 기준 체크 (3개)** — 각 항목 적합/부분 적합/미적합 판정
   - 직무 적합 — 내 경력이 이 역할을 실제로 수행할 수 있는가
   - 환경 적합 — 이 회사의 규모·문화가 내 Must 조건과 맞는가
   - 성장 적합 — 이 포지션이 내가 원하는 경험을 줄 수 있는가
2. **화장품 경력 필수 감지** — JD에 "화장품 경력 필수" 문구 있으면 자동 경고
3. **판정** — 지원 추천 / 검토 필요 / 패스
4. **어필 포인트 3개** — JD에 맞게 재구성된 강점 (개별 복사 가능)
5. **이력서 커스텀 가이드** — 한 줄 출력
6. **이력서 초안 자동 생성** — 자기소개 + 핵심 경력 3줄 + 지원 동기 (복붙 가능 수준)

### 판정별 다음 행동
- 지원 추천: 어필 포인트 전체 복사 / 새 JD 분석
- 검토 필요: 어필 포인트 미리 복사 / 새 JD 분석
- 패스: 새 JD 분석 / "그래도 지원할게요" 링크

---

## Tab 3 · 지원 현황 (CRM)

분석한 공고를 보드 형태로 관리.

**필터 탭** — 지원 추천 / 검토 필요 / 패스 / 전체 (각 건수 표시)

**컬럼** — 회사명 / 직무 / FIT / 판정 / 상태 / 날짜

**상태 드롭다운** — 지원 전 / 서류 / 면접 / 최종 / 패스

**메모** — 공고별 메모 추가

분석할 때마다 자동 저장.

---

## 하네스 + 오케스트레이션

이 서비스의 핵심 작동 원리.

**하네스 (말의 고삐)** — Claude가 딴 길로 안 가게 규칙 고정.
시스템 프롬프트에 핵심 판단 기준(직무·환경·성장 적합) 고정 +
화장품 경력 필수 감지 규칙 고정. 기준이 고정돼야 공고 간 비교가 가능.

**오케스트레이션 (지휘)** — 할 일의 순서를 설계.
JD 입력 → 기준 체크 → 화장품 감지 → 판정 → 어필 → 이력서 가이드.
순서가 설계되면 공고만 넣어도 한 번에 결과가 나온다.

## 기술 스택

- 단일 HTML + Vanilla JS + Tailwind CDN
- Claude API (claude-sonnet-4-20250514) 실시간 호출
- localStorage 데이터 유지
- 폰트 Pretendard
- **다크모드** (베이지 톤에서 판정 배지 구분이 약해 전환)

### localStorage 구조
```
careerfit_api_key   → "sk-ant-..."
careerfit_profile   → { reason, resume_raw, summary, strengths[], must[], keywords[], updated_at }
careerfit_history   → [{ id, company, position, verdict, verdict_reason, appeals[], gaps[], analyzed_at }]
```
