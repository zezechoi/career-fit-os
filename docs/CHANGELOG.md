# Career FIT OS — 변경 이력

> 최초 기획서(`career_fit_prompt.md`) 대비 현재 구현까지의 변경사항.
> 1차 출처: 3주차 미션 회고(`3주차 미션 Career FIT OS.md`, 최지예 작성).

---

## 변하지 않은 핵심 (정체성)

- **철학** — 나도 회사를 고른다 · 서로 맞아야 오래간다 · 최종 결정은 내가 한다
- **패스해도 지원 가능** — 판정이 부정적이어도 사용자가 진행 선택 가능
- **이력서 1회 입력 → 이후 공고만** — 온보딩 1회, 반복은 공고만
- **1인용** — 고객은 본인 한 명. 범용화 안 함

---

## 주요 변경사항 (시간순)

### 1. 첫 구현 — 전부 "보류" 판정 문제
공고를 넣으면 다 "보류"가 떴음. 원인은 판단 기준이 틀렸던 것.
"브랜드 기획 직무냐"로 잡혀 있었는데 본인 경력과 안 맞는 기준이었음.
→ 더 근본 문제 발견: 판단 도구 전에 판단 기준이 먼저였다.

### 2. 기준 재설정 — 본인 분석부터 다시
해외영업 6년 + IT 마케터 1년, 화장품 경력 없음을 기준으로
온보딩 판단 기준을 4개로 교체.

### 3. 핵심 기준 4개 → 3개 통합
실제로 쓰면서 직무 적합 / 환경 적합 / 성장 적합 3개로 통합·정교화.
(기존 2·4번이 "브랜드 전체 흐름"으로 중복됐던 것을 해소)
네이밍도 "4대 기준 체크" → "핵심 기준 체크".

### 4. 화장품 경력 필수 감지 추가
화장품 경력이 없으니 JD에 "화장품 경력 필수" 문구 있으면 자동 경고.
처음엔 뺄까 고민했으나 — 지원 못 할 공고를 미리 걸러주는 시간 절약 기능으로 판단.

### 5. 이력서 초안 자동 생성 추가
공고 판단만으로 부족. 괜찮은 공고 찾아도 이력서 맞추는 걸 매번 새로 고민해야 했음.
JD 기반으로 자기소개 / 핵심 경력 3줄 / 지원 동기 초안 자동 생성. 복붙 가능 수준.

### 6. 구조 개편 — 2탭 → 3탭
처음엔 온보딩 / JD 분석기 2탭. 분석 기록이 섞여 복잡했음.
프로필 / 공고 판단 / 지원 현황 3개로 분리하면서 흐름이 명확해짐.

### 7. 다크모드 전환
밝은 베이지 톤(#F8F5EC)에서 판정 배지 구분이 약했음.
어둡게 바꾸니 지원 추천(초록) / 검토 필요(노랑) / 패스(회색)가 확실히 구분됨.

### 8. FIT 점수 체계 도입
정량 점수 추가. (초기엔 전부 100으로 나와 변별력 부족 → 재설계 중)

## 진행 중 / 예정 작업

- [ ] FIT 점수 체계 재설계 (3개 기준 결과를 점수에 실제 반영)
- [ ] 기존 저장된 분석 결과를 새 3개 기준으로 재분석 (소급 적용)
- [ ] 지원 현황 FIT 컬럼 정렬 기능 (높은 순/낮은 순 토글)
- [ ] 회사명·직무명 말줄임표 호버 툴팁
- [ ] 날짜 컬럼 레이블 명확화 (분석일/마감일)
- [ ] 지원 현황 행별 삭제 + 일괄 삭제 기능
- [ ] 핵심 기준 판단 근거 출처 태그 ([JD 기반]/[프로필 기반]/[추정])
