# CLAUDE.md — Career FIT OS

이 파일은 Claude Code가 이 프로젝트를 작업할 때 매번 따라야 할 고정 지침이다.

---

## 이 프로젝트는 무엇인가

이직 판단 도구. 공고(JD)를 넣으면 판정 → 어필 포인트 → 이력서 초안까지 자동으로 나온다.

**현재 단계: 소규모 멀티유저 테스트.**
원래 제작자(최지예) 1인용으로 출발해 MVP까지 완성했고,
지금은 몇 명이 함께 테스트할 수 있도록 확장하는 중이다.
검증되면 더 넓게 열 수 있으나, 그 전까지는 "가볍게 여는 것"이 목표다.

- Live: https://career-fit-os.vercel.app/
- 스택: 단일 HTML + Vanilla JS + Tailwind CDN + Claude API + localStorage

---

## 하네스 (지켜야 할 원칙)

1. **각자의 키, 각자의 비용** — 사용자는 본인 Claude API 키를 직접 입력한다.
   서비스가 키를 대신 관리하거나 비용을 부담하는 구조로 바꾸지 마라.
   (결제·과금 시스템은 이 단계에서 만들지 않는다.)

2. **판단 기준은 사용자별로 설정 가능** — 직무 적합 / 환경 적합 / 성장 적합 3개는
   **기본 템플릿**이다. 사용자가 온보딩에서 자기 기준으로 바꿀 수 있어야 한다.
   특정 산업(K-뷰티 등)에 기준을 하드코딩하지 마라.

3. **사용자 데이터는 분리** — 한 사용자의 프로필·히스토리가 다른 사용자에게
   보이면 안 된다. 데이터 격리를 항상 우선한다.

4. **판정 = 결정 보조, 강제 아님** — "패스" 판정이어도 "그래도 지원" 경로를 항상 남긴다.
   OS는 판단을 돕고, 결정을 막지 않는다.

5. **단순함 우선** — 소규모 테스트 단계다. "만드는 시간 > 쓰는 시간"이 되는
   과한 기능(복잡한 권한 관리, 관리자 대시보드 등)은 경고하라.

## 오케스트레이션 (실행 순서)

JD 입력 → 핵심 기준 체크 → 판정 → 어필 포인트 → 이력서 가이드 → 이력서 초안

이 순서를 바꾸거나 단계를 건너뛰지 마라.
(주의: '화장품 경력 필수 감지'는 제작자 본인용 기능이었다. 범용화하면서
산업 중립적으로 바꾸거나, 사용자가 켜고 끌 수 있는 옵션으로 일반화하라.)

## 보안 주의 (멀티유저에서 중요해짐)

- 사용자 API 키는 절대 코드·로그·다른 사용자에게 노출되면 안 된다.
- 한 사용자가 다른 사용자의 데이터에 접근할 수 없도록 항상 검증하라.
- 키 입력 시 평문 노출, 콘솔 로그 출력 등을 하지 마라.
