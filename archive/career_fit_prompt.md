# Career FIT — Claude Code 프롬프트

---

## 한 줄 목표

"나와 이 회사가 맞는지 빠르게 판단해서 시간을 아끼는 것"

---

## 만들 것

단일 HTML 파일 1개. 탭 2개.
Vanilla JS + Tailwind CDN.
Claude API (claude-sonnet-4-20250514) 실시간 호출.
localStorage로 데이터 유지.

---

## 철학 — 코드 전체에 녹아들어야 함

- 나도 회사를 고른다
- 서로 맞아야 오래간다
- 최종 결정은 내가 한다

판정이 "패스"여도 사용자가 "그래도 지원" 선택 가능.
OS는 판단을 돕고, 결정을 막지 않는다.

## Tab 1 · 온보딩

### 상단 철학 텍스트 (항상 보임, 작게)
"나도 회사를 고른다 · 서로 맞아야 오래간다 · 최종 결정은 내가 한다"

### Step 1 — 이직 이유 선택
질문: "이직을 결심한 가장 큰 이유가 뭔가요?"
버튼 탭 방식, 하나만 선택:
- 연봉을 높이고 싶어서
- 성장이 막혀서
- 회사/팀 문화가 안 맞아서
- 하고 싶은 직무가 달라서
- 기타 (직접 입력)

### Step 2 — 이력서 입력

입력 방식 (탭 전환):

텍스트 붙여넣기 (기본):
입력창에 샘플 텍스트 미리 채워져 있음 (아래 참고).
사용자가 지우고 본인 이력서 붙여넣으면 됨.

샘플 텍스트:
  OO회사 · 마케팅팀 · 2022~현재
  - 글로벌 브랜드 SNS 채널 운영
  - 해외 파트너사 커뮤니케이션
  - 콘텐츠 기획 및 캠페인 집행

  OO회사 · 영업팀 · 2019~2022
  - 신규 거래처 발굴 및 계약
  - 해외 전시회 바이어 상담
  - 제안서 작성 및 수주

파일 업로드 (PDF/DOCX):
DOCX → mammoth.js CDN으로 텍스트 추출 후 textarea에 자동 채움.
PDF → pdf.js CDN으로 텍스트 추출 후 textarea에 자동 채움.
파일을 API로 직접 보내는 게 아니라 텍스트 변환 후 동일하게 처리.
실패 시: "파일을 읽지 못했어요. 텍스트를 직접 붙여넣어 주세요."

공개 링크 (노션 등):
fetch로 텍스트 추출 시도.
실패 시: "링크에서 내용을 읽지 못했어요. 텍스트를 직접 붙여넣어 주세요."

"분석 시작" 버튼 → Claude API 호출

### API 호출 — 온보딩

system:
  커리어핏은 이직 준비를 돕는 커리어 전문가입니다.
  핵심 원칙: 지원자도 회사를 고릅니다. 서로 맞아야 오래갑니다.
  판단은 돕되, 결정은 본인이 합니다.
  반드시 JSON만 응답하세요. 설명, 마크다운, 코드블록 없이 순수 JSON만.

user:
  이직 이유: [reason]
  이력서: [resume]

  아래 형식의 JSON만 응답해줘:
  {"summary":"한 줄 포지셔닝 20자 이내","strengths":["강점1","강점2","강점3"],"must":["Must조건1","조건2","조건3"],"keywords":["키워드1","키워드2","키워드3","키워드4","키워드5"]}

JSON 파싱:
  const text = data.content[0].text.replace(/```json|```/g, '').trim();
  const profile = JSON.parse(text);

### 온보딩 결과 화면

- 포지셔닝 한 줄 (크게)
- 핵심 강점 3개
- Must 조건 3개
- "수정하기" 버튼 → 인라인 편집 가능
- "저장하고 시작하기" 버튼 → Tab 2로 자동 이동

저장 위치: localStorage("careerfit_profile")
언제든 "내 프로필 수정" 버튼으로 재분석 가능.

저장된 포지셔닝 한 줄 + Must 조건 3개 작게 표시.
"프로필 수정" 링크.

### JD 입력

입력 방식 (탭 전환):

이미지 업로드 (기본 — 메인):
잡코리아 등 텍스트 복사가 안 되는 공고 대응.
스크린샷, 캡처, 사진 모두 가능.

텍스트 붙여넣기 (보조):

Placeholder: "채용공고 텍스트를 붙여넣으세요"

"분석하기" 버튼

### API 호출 — JD 분석 (텍스트)

system:
  커리어핏은 이직 준비를 돕는 커리어 전문가입니다.
  핵심 원칙: 지원자도 회사를 고릅니다. 서로 맞아야 오래갑니다.
  판단은 돕되, 결정은 본인이 합니다.
  반드시 JSON만 응답하세요. 설명, 마크다운, 코드블록 없이 순수 JSON만.

user:
  내 프로필:
  포지셔닝: [profile.summary]
  핵심 강점: [profile.strengths]
  Must 조건: [profile.must]
  역량 키워드: [profile.keywords]

  채용공고: [jd]

  아래 형식의 JSON만 응답해줘:
  {"company":"회사명(없으면 미상)","position":"포지션명","verdict":"지원 또는 보류 또는 패스","verdict_reason":"판정 이유 2문장. 내 Must 기준 + JD 요건 기준 양방향으로.","appeals":["어필1 — 내 경력 증거 포함 완성 문장","어필2","어필3"],"gaps":[{"gap":"부족한 것","strategy":"이렇게 대응하면 됩니다"}]}

### API 호출 — JD 분석 (이미지)

messages content에 image + text 함께 전달:

  messages: [{
    role: "user",
    content: [
      {
        type: "image",
        source: {
          type: "base64",
          media_type: "image/jpeg",
          data: base64ImageData
        }
      },
      {
        type: "text",
        text: "내 프로필:\n포지셔닝: [summary]\n핵심 강점: [strengths]\nMust 조건: [must]\n역량 키워드: [keywords]\n\n이 채용공고 이미지를 분석해서 아래 JSON으로만 응답해줘:\n{...텍스트와 동일한 JSON 형식}"
      }
    ]
  }]

JSON 파싱:
  const text = data.content[0].text.replace(/```json|```/g, '').trim();
  const result = JSON.parse(text);

## 에러 처리

- API 키 없음 → API 키 입력 화면으로
- API 호출 실패 → "분석 중 오류가 생겼어요. 다시 시도해주세요" + 재시도 버튼
- JSON 파싱 실패 → 코드블록 제거 후 재파싱. 그래도 실패 시 재시도 버튼
- 이력서 미입력 → "이력서를 먼저 입력해주세요" + 온보딩 탭 이동 링크
- 파일 변환 실패 → "파일을 읽지 못했어요. 텍스트를 직접 붙여넣어 주세요"
- 링크 실패 → "링크에서 내용을 읽지 못했어요. 이미지나 텍스트로 입력해주세요"
- 로딩 중 → 스피너 + "분석 중..." 텍스트. 버튼 비활성화.

## 빌드 순서

1. API 키 입력 + localStorage 저장
2. 온보딩 탭 — 이직 이유 선택 + 텍스트 이력서 입력 + AI 분석 + 결과 확인/수정
3. JD 분석기 탭 — 이미지 업로드 (base64 + Claude 비전) ← 메인
4. JD 분석기 탭 — 텍스트 붙여넣기 (보조)
5. JD 분석 결과 화면 — 판정 + 어필 + 갭
6. 판정별 다음 행동 버튼
7. 히스토리 자동 저장 + 리스트
8. 히스토리 클릭 → 인라인 결과 + 뒤로가기
9. 이력서 파일 업로드 (mammoth.js, pdf.js)
10. 이력서 링크 입력 옵션
